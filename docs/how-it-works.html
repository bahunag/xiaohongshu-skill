你跟 Claude（AI 助手）说"帮我分析张咋啦的近30天笔记"，接下来发生的事情，就像你让一个朋友帮你做事——他有自己的工具箱，知道该用哪把工具。

02

技术架构：从上到下，分层读懂

整个系统像一栋楼，每层各司其职，信息从顶层传下来，结果从底层传回去。把鼠标放到每层上，感受一下每层的作用。

🏠 第一层 · 顶层
你（使用者）
整个系统的起点。你用中文说一句话，比如「帮我看看 AI 相关的热门笔记」。不需要懂代码，就像你给朋友发微信一样。
自然语言 中文指令
语言理解 · 意图识别
🧠 第二层
Claude（AI 大脑）
Anthropic 公司开发的 AI。它理解你说的话，判断需要做什么，然后查看自己有哪些工具可以用。它是整个系统的"大脑"和"调度员"。
语言理解 任务规划 工具调度
读取说明书 · 决定用哪个工具
📖 第三层
小红书 Skill（操作手册）
你现在看的这个 Skill，本质上是一个 Markdown 文档，里面写着"遇到小红书相关的请求，应该怎么做、有哪些工具、怎么调用它们"。Claude 读完它，就知道了该做什么。
SKILL.md 文档 工具说明 触发条件 工作流程
HTTP 协议 · MCP 格式
⚙️ 第四层
xiaohongshu-mcp（执行引擎）
一个运行在你电脑上的小程序（端口 18060）。它是真正干活的那个——收到指令后，打开一个看不见的浏览器，模拟人的操作，登录小红书、搜索、翻页、提取数据。用 Go 语言写的，速度很快。
go-rod 浏览器引擎 端口 :18060 Cookies 登录
模拟人工操作浏览器
🌐 第五层 · 底层
小红书（数据来源）
最终的数据来源。程序以你的账号身份访问小红书网站，取到点赞数、收藏数、笔记内容等真实数据。你得先扫码登录，才能取到数据。
真实用户数据 需要登录状态 Cookies 验证
03

功能列表：能做什么，不能做什么

现在上线了 7 个功能，6 个还在开发中（标了🔒的那些暂时不能用）。

🔐
check_login_status
检查你是否还登录着。就像看手机屏幕左上角有没有 WiFi 信号，一眼知道能不能用。
📱
get_login_qrcode
生成一个二维码图片，你用小红书 App 扫一下，就登录上了。跟扫码进入朋友分享的链接一样。
🔍
search_feeds
搜索关键词，找笔记。可以按「最多点赞」「最新」等方式排序，还能筛选时间段和内容类型。
🏠
list_feeds
获取小红书首页给你推荐的内容——和你打开 App 看到的一样，反映的是你账号的兴趣标签。
👤
user_profile
获取某个账号的主页信息：粉丝数、获赞数，以及他/她发过的所有笔记列表。
📄
get_feed_detail
打开某篇笔记的完整内容：正文、图片、精确的点赞数/收藏数/评论数，以及评论列表。
🗑️
delete_cookies
清除登录状态，退出账号。就像手机上点「退出登录」一样。
💬
post_comment_to_feed
在笔记下面发评论。
↩️
reply_comment_in_feed
回复某条评论。
❤️
like_feed
给笔记点赞或取消点赞。
⭐
favorite_feed
收藏或取消收藏笔记。
📸
publish_content
发布图文笔记，支持定时发布。
🎬
publish_with_video
发布视频笔记。
🔒
标了「待开发」的6个功能目前不可使用，调用会报错。它们涉及往账号里写入数据（发评论、点赞、发笔记），需要更严格的风控测试，后续版本会陆续开放。
04

各部分之间是怎么"传话"的

每次你提出请求，背后会发生一系列"传话"——就像一个工作群里，同事之间把任务层层传下去，再把结果传回来。

🔗 技术层面：一次完整请求的通信链路
你
帮我搜一下「AI工具」最近最火的笔记
你
AI
Claude → Skill 文档
我去看看 SKILL.md 说怎么做……好，它说用 search_feeds 工具，参数是 keyword="AI工具", sort_by="最多点赞"
📖
Skill 文档 → MCP 服务
发送 HTTP 请求到 localhost:18060/mcp，格式是 JSON，带上 Mcp-Session-Id 身份凭证，请求调用 search_feeds 工具
⚙️
MCP 服务 → 小红书网站
打开隐身浏览器，加载 Cookies 登录状态，访问 xiaohongshu.com，模拟用户搜索"AI工具"，滚动页面抓取结果
🌐
小红书 → MCP 服务
返回搜索结果，包含每篇笔记的 id、标题、作者、封面图、点赞数
⚙️
MCP 服务 → Claude
把网页里的信息整理成干净的 JSON 格式返回，22条结果，第一条是「deepseek这套组合拳，别人以为我开挂了」，点赞 11.9万
AI
Claude → 你
我找到了22条结果，帮你整理好了 Top 5，最火的是……（以下是完整报告）
🔑
Cookies 是什么，为什么这么重要？
Cookies 就是你的"登录通行证"，存在一个叫 cookies.json 的文件里。MCP 服务每次模拟你访问小红书时，都要带着这个通行证，否则小红书会把它当成机器人，拒绝访问。如果提示「未登录」，就是这个通行证过期了，需要重新扫码刷新。
05

用到了哪些第三方服务

这个 Skill 站在几个已有工具的肩膀上。以下是它们分别是什么、做什么用的。

🤖
Claude（Anthropic）
AI 大脑
Anthropic 公司做的 AI 助手。它负责理解你说的话、规划做事步骤、调用工具、整理结果。整个系统的"指挥官"。
⚙️
xiaohongshu-mcp
专用浏览器机器人
一个开源程序（Go 语言写的），专门为小红书设计。它能打开一个"隐身浏览器"，模拟真实用户操作，把网页数据整理成 AI 能读懂的格式。
🕹️
go-rod
浏览器遥控器
xiaohongshu-mcp 内部用的工具库。就像一个遥控器，能控制浏览器点击、滚动、等待、截图……你看不到它，但它在默默干活。
🔌
MCP 协议
AI 和程序之间的"普通话"
Model Context Protocol，Anthropic 制定的一套规范。有了它，AI 可以跟各种外部程序对话，就像手机充电口统一成 USB-C 一样，大家说同一种语言。
🐍
Python
中间层脚本语言
Claude 用 Python 代码来发请求给 MCP 服务。就像写一张快递单，指定要寄什么、寄到哪、用什么方式。这些代码你不需要写，Claude 自动生成。
🌐
小红书（数据来源）
数据仓库
最终所有笔记、点赞数、评论都来自小红书的真实网站。这个 Skill 只是帮你"读"数据，不会绕过任何安全限制，以你自己的账号身份访问。
06

从你说话到拿到数据，每一步发生了什么

完整的时间线，从头到尾拆开来看。每步只需要零点几秒，整体30秒内完成。

第 1 步
你说话 → 用中文打出你的请求，Claude 收到
第 2 步
Claude 识别意图 → 判断这是小红书相关请求，找到「小红书 Skill」文档
第 3 步
Claude 读取 Skill → 知道有哪些工具可用、调用参数是什么
第 4 步
发 HTTP 请求 → Python 代码向 localhost:18060/mcp 发送初始化请求，拿到 Session-Id
第 5 步
调用工具 → 带上 Session-Id 再发一条请求，指定工具名称和参数（比如 keyword="张咋啦"）
第 6 步
MCP 开始执行 → 程序打开隐身浏览器，加载你的 Cookies，访问小红书网站
第 7 步
抓取数据 → 模拟人工操作：输入关键词、等待页面加载、读取每个数字
第 8 步
整理格式 → 把网页 HTML 转换成干净的 JSON 格式（结构化数据），关闭浏览器
第 9 步
返回给 Claude → Claude 收到 JSON 数据，理解其中的含义
第 10 步
Claude 输出 → 把数据整理成你看得懂的中文报告，展示在对话框里
⏱️
整个10步流程，单次工具调用大约 3～8秒。分析一个账号需要多次调用（搜索 + 获取主页 + 逐篇详情），总时间约 30秒～2分钟，取决于笔记数量。
